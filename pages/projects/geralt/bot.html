    Who is Geralt?

    A simple Discord bot written in Python using the discord.py API wrapper. Named after the protagonist Geralt of Rivia. On the contrary, there is nothing related to the Witcher in this bot. Geralt provides a handful of easy-to-use yet robust commands, making your life on Discord much easier.

    Started off as a small project to impress my friends from school which then led to a full-fleged hobby to develop the bot with many features and become a very good programmer. Till date, Geralt is being maintained and improved. The following will give you proper instructions for setting up the bot if you are going to run your own instance.

    The default prefix is .g

    Setup

    Clone the repository using git, if you're to lazy to go to github, here it is - git clone https://github.com/bsod2528/Geralt.git. Then, you will be needing:

  • Python 3.8 or more
  • PostgreSQL
  • And a bunch of packages - requirements.txt

    • PostgreSQL

    You will be needing the following:

  • database named geralt.
  • role named av with maximum permissions.
  • and a bunch of tables - tables.sql

    • Python Env.

    Create an environment by running py -m venv venv and install dependencies in requirements.txt inside a folder called Geralt.

    Running Geralt

    Once all the above requirements are met, you will be able to run your very own instance of Geralt. To run him, simply double-click the geralt.py on your computer if you're locally hosting else run the necessary command to run if its on a vps.

    What is a Cog?

    A "Cog" is the baseclass for all commands of a certain category to be present in. Consider it like a category of commands, and each command would be the method inside it. In Geralt, I've created a total of 11 cogs in which 2 are listeners. We shall now dive right into them.
