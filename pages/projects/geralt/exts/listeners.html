    • Listeners

    Function decorators which ultimately act as listeners to certain events that occur on Discord on a regular basis. Now by default, the bots don't do this, we have to give them that power, i.e., program them in such manner.

    Geralt listens to 4 major types of events:

  • Errors
  • Messages
  • User updation
  • Guild joining/leaving

    • Events

    Messages

    Events such as on_message_delete and on_message_edit are heard by Geralt. From this, I store only a portion of the information and not on the whole for making sure that the snipe commands work properly.

    Highlight mainly depends on these events! For the conversion of emote urls in a message into a webhook message, this is required.

    User updation

    On user updating their account, Geralt logs the changes made to their avatar, username, and discriminator. Users have to opt - in for enabling this functionality. This can be done by runnning the userlog commands.

    Guild joining/leaving

    For a personal note I log which guilds Geralt joins and leaves. This is done by letting him listen to on_guild_join and on_guild_leave events. However, these are only logged in a channel only when Geralt is on as he is a locally hosted bot.

    ErrorHandler

    This is a cog which has only one single listener running, which is the on_command_error. This event is the gateway to all the different types of errors that may occur upon invoking a command.

    I check whether if the user is not an owner, if there was command not found, application command errors, concurrency errors, missing permissons, member not found, wrong argument, cooldown, missing argument, and at last, errors which were not handled in the above mentioned using a simple else statement.
