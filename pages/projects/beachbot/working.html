    • Bot Subclass

    Learn more about classes here: Classes

    Refer the documentation for the classes and methods mentioned accordingly: discord.py docs

    Inside discord.ext.commands there is a class called Bot. We first subclass this and name it as BaseBeach. The init method consists of all the basic parameters the bot requires to function. Upon passing the parameters, we pass onto the next method.

    async def load_exts - This method loads all the cogs in an async manner from the list of all the directories of our cogs. Yet, it is not executed.

    The setup_hook method is where all the bot loading tasks are supposed to be executed, i.e., loading cache, connecting to the database, loading cogs, etc. This method is executed before the bot it is connected to the Websocket (before it is online).

    We then listen to the on_ready event which checks if the bot is ready to be used. On ready, we print in the console that "Beach-Bot is online!".

    Running the Bot

    The bot-subclass, and others are present inside bot.py Outside this class, we have an async function called run. We instantiate the subclass and run start method. Yet, this function is not called. In the file called launcher.py which is outside the beach folder calls this function, thus the bot runs.

    In bot.py, we import dotenv library which can be installed from pip which loads the config.env file which stores all my credentials needed. In this case, the bot's secret token.

    Oh, for your information, you need the following requirements installed in your Python environment in order to run Beach-Bot successfully. You can check them out right here: requirements.txt

    Cogs

    A cog is a combination of commands which are related to each other by purpose and target. For example, Moderation cog - will have all the commands related to moderation activities such as ban, kick, mute, etc.

    Each cog is a class containing methods for the sake of easy development and organisation. Commands are present as a method inside the class.

    There are a total of 4 cogs present. Developer, Fun, ErrorHandler, and Help are the cogs present. Developer contains commands for the bot owner to use, Fun has few commands which induce dopamine and make the user happy. ErrorHandler is for handling all the errors that occur upon running a command, and at last the Help cog is a subclass of commands.MinimalHelpCommand which takes care making the default help command much more beautiful.

    We shall now see the commands present in each cog. The syntax alone for each command will be given. A detailed description on creating cogs and commands will be mentioned in the next page.

    Commands

    A command is basically a method in a cog class which is defined using the @command decorator. We will be learning more on this in the next section don't worry!

    The Developer cog has only three commands which are load, unload, and reload. These commands take care of making changes to a command when made without actually having to restart the entire bot.

    The Fun cog has 5 commands, in which 2 are text, 1 is a slash command and the other two are context menu commands. Out of these context menu commands, one is a user based and the other is a message based context menu.

    In the comming sections, you will be learning on how these commands work, how to make them and etc.
